## Issues Fixed

### GeoZones.tsx — 14 errors (cascading from one root cause)

**Root cause:** `Map` was imported from `lucide-react` without an alias, shadowing
the global JavaScript `Map` constructor. Every subsequent `new Map<K, V>()` call
inside the file resolved to the Lucide icon type instead of the built-in collection
class, causing a cascade of type failures.

**Fixes applied:**
- `import { Map, ... }` → `import { Map as MapIcon, ... }` (line 8)
- Updated two usage sites: `icon: Map` → `icon: MapIcon` (lines 949, 1089)
- Added missing `offline: '#9ca3af'` key to `TIER_COLORS` record (line 132)
  — `ZoneTier` union includes `'offline'` but the record only had 5 of 6 keys.

### FuelTheft.tsx — 2 errors

**Fix 1:** `getSensor()` computed `dropPct` locally but did not include it in the
returned object. The call site used `sd.dropPct ?? Math.round(...)` which was always
falling through to the fallback anyway. Removed the non-existent property reference;
the calculation is now used directly.

**Fix 2:** Inline style used `ringColor` which is not a valid CSS property (and causes
a strict TS error). Replaced with `'--tw-ring-color'` (the Tailwind custom property
that actually controls `ring-*` utilities) and cast the object as `React.CSSProperties`.
